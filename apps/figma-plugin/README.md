# Figma Plugin MVP

This plugin imports the `.scene.json` files generated by the Chrome extension and recreates them as Figma layers.

Current support:

- frame nodes
- text nodes with font fallback
- image nodes with remote image loading
- background colors
- simple borders
- simple corner radius
- simple drop shadows

Current limitations:

- no SVG vector reconstruction yet
- no pseudo-elements
- no screenshot fallback blocks yet
- no true auto layout reconstruction yet
- background images are best-effort only
- remote images can still fail if the source blocks plugin access or serves unsupported formats

## Load it in Figma

Use the Figma desktop app. The current Figma quickstart docs still require the desktop app for local plugin development.

1. Open a Figma design file in the desktop app.
2. Open the Development menu.
3. Choose `Import plugin from manifest...`
4. Select `/Users/cristian/Documents/GitHub/web-to-figma/apps/figma-plugin/manifest.json`

If Figma asks you to use a generated plugin ID, create a local plugin once from Figma's `New plugin...` flow, then replace its files with the files from this folder.

## Use it

1. Capture a page in the Chrome extension.
2. Either save the `.scene.json` file or use `Capture + Copy JSON`.
3. Run `Web to Figma Import` in Figma.
4. Paste the JSON or load the saved file.
5. Click `Import into Figma`.

## Why the manifest allows all domains

The plugin uses Figma image-loading APIs against arbitrary website image URLs captured by the Chrome extension. Because those image hosts vary per website, the manifest currently uses `"allowedDomains": ["*"]` with a required explanation string.
